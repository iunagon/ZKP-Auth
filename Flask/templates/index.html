ZKPAuth

Password based authentication system is the most commonly used authentication system for most of the web applications in todayâ€™s world. This system uses cryptographic principles such as hashing making it difficult for the hackers to crack the passwords. These password hashes are often stored in a centralized database and hence are vulnerable to get stolen. Hackers can steal these hashes from the server on which they are stored and try to crack the passwords using methods such as Dictionary Attacks, Brute Force Attacks, Lookup Tables, Reverse Lookup Tables and Rainbow tables. To avoid this need to store sensitive information for authentication, we introduce the concept of zero knowledge proof.

Zero knowledge proof (ZKP) is a type of proof in which one party (prover) proves another party (verifier) that he/she possess certain information without actually revealing any of the information. With this we will be able to authenticate the user, without the need for them to reveal any sensitive information, in this case the password. So now, as even the server does not possess the hashes of the passwords or any information related to the passwords, the hackers will not be able to steal any sensitive information from it. Hence, improving the security of the authentication process.
